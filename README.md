This week focused on extending Organicus, a mobile app for browsing and purchasing organic food products, by designing a complete checkout and payment experience. The goal was to ensure users can move smoothly from product selection to purchase completion in a simple, secure, and trustworthy flow.

⸻

Key Focus Areas

* Structuring a clear and intuitive checkout flow
* Designing a clean and minimal payment interface
* Maintaining strong visual hierarchy for better readability
* Reinforcing user trust through confirmation and feedback states
* Ensuring consistency with the Organicus brand (colors, typography, spacing)

⸻

Screens Designed

1. Checkout Page

The checkout screen acts as a summary and decision point before payment.

Features include:

* Product summary (image, product name, and price)
* Quantity display for selected items
* Delivery details section (address and order information)
* Clear cost breakdown (subtotal, delivery fee, total)
* Primary CTA: “Proceed to Payment”

The layout was designed to help users quickly review their order without feeling overwhelmed. Spacing and hierarchy were used to clearly separate key information.

⸻

2. Add Payment Page

This screen focuses on simplicity and trust, allowing users to enter payment details with ease.

Features include:

* Multiple payment options (e.g., card, mobile payment where applicable)
* Clean input fields for card details (name, number, expiry date, CVV)
* Minimal distractions to keep focus on completing the transaction
* Primary CTA: “Pay Now”

Special attention was given to spacing, alignment, and form structure to make the process feel secure and effortless.

⸻

3. Order Confirmation Screen

The confirmation screen provides feedback and reassurance after a successful transaction.

Features include:

* Success message (e.g., “Order Successful”)
* Brief order summary
* Visual feedback (icon or illustration)
* CTA: “Continue Shopping”

This screen reinforces trust and gives users a clear sense of completion, which is essential in any purchase flow.

⸻

Design Approach

The design approach focused on creating a smooth, frictionless, and guided experience. Each screen was structured to move the user step by step:

Product Selection → Checkout Review → Payment → Confirmation

Consistency with previous Organicus screens was maintained through:

* A consistent color palette (clean, fresh tones)
* Unified typography and spacing system
* Reusable UI components such as buttons and layout patterns

The goal was to ensure the checkout experience felt like a natural continuation of the app, not a separate or disconnected flow.

⸻

Tools Used

* Figma

⸻

Outcome

The addition of the checkout and payment flow completes the end-to-end user journey within Organicus. Users can now move from browsing organic products to successfully placing an order in a clear, simple, and visually consistent experience.

This project demonstrates a stronger understanding of user flow design, transaction-based interfaces, and designing for usability and trust—moving beyond static screens toward a more complete product experience.
